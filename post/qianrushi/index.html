嵌入式系统概述-微处理器

  1. 能否简述CPU内部控制部件、运算部件、寄存器的主要作用？
    • CPU内部控制部件：主要负责取指、译码和取操作数等基本功能，并发送主要的控制指令。
    • 运算部件：算术运算单元，进行数学运算，如加、减、出或数值的比较；另一部分是逻辑运算单元，主要处理逻辑运算工作，如AND、OR、XOR或NOT等运算。
    • 寄存器：用于储存暂时性的数据。
  2. 能否简述3-5点嵌入式处理器的主要特性？
    • 体积小、集成度高、价格较低
    • 可扩展的处理器结构
    • 功耗低
    • 对实时多任务有很强的支持能力
    • 具有功能很强的存储保护功能
  3. 能否试着分析微处理器、微控制器、DSP和SOC的特点、区别及用途？
    • 微处理器：由通用计算机中的CPU演变而来，装配在专门设计的电路板上，只保留和嵌入式应用紧密相关的功能硬件，去除其它冗余的功能，以最低功耗和成本实现特定的需求。为了满足嵌入式应用的特殊要求，在工作稳定、抗电磁干扰、可靠性等方面对CPU做了各种增强。
    • 微控制器：又称单片机，以某一种微处理器内核为核心，特点是单片化、体积大大减小，从而使功耗和成本下降、可靠性提高
    • DSP：系统结构采用Harvard结构和专用硬件乘法器、指令系统采用快速DSP指令，适用于处理器运算速度较高、向量运算较多的应用领域、
    • SoC片上系统：一种系统集成芯片，功能可以完全由硬件实现，也可也由软硬件完成。高密度、高速度、高抗干扰性
  4. 能否试着分析一款SOC处理器的内部软件？
    • 麒麟970CPU部分采用的仍然是ARM公版A73架构+A53架构大小核心搭配，4个A73大核最高主频为2.4GHz，4个A53小核最高主频为1.8Ghz；GPU方面，麒麟970采用了ARM Mali-G72MP12，这是一款12核的图形处理单元；继承了NPU(Neural network processing Unit)神经网络单元，官方宣称像将于四个Cortex-A73核心，处理想用AI任务，新的异构计算架构拥有约50倍能效和25倍性能优势，图像识别速度可达到约2000张/分钟。
  5. 能否试着调研ARM、MIPS、POWER-PC的主要市场？
    • ARM：Cortex A系列处理器：高端嵌入式应用市场；经典ARM及Cortex R系列处理器：中高端嵌入式应用市场；Cortex M系列处理器：低端嵌入式应用市场
    • MIPS：应用于消费类电子、下一代网络、宽带产品、智能卡、机顶盒、数字电视、DVD
    • Power-PC：应用于DSL调制解调器、SOHO路由器、远程接入服务器、DSLAM、执行局交换机设备、无线基站、企业路由器

嵌入式系统概述-嵌入式系统

  1. 能否简述嵌入式操作系统的体系结构及其各个层次的功能？
    • 驱动层：对操作系统和应用提供所需的驱动支持
    • 硬件抽象层：通过特定的上层接口与操作系统进行交互，屏蔽了底层硬件的多样性
    • 操作系统层：包括嵌入式内核、嵌入式TCP/IP网络系统、嵌入式文件系统、嵌入式GUI系统和电源管理等部分。
    • 中间件层：在一些复杂的嵌入式系统中也开始采用中间件技术，主要包括嵌入式CORBA、嵌入式Java、嵌入式DCOM和面向应用领域的中间件软件。
    • 应用层：由多个相对独立的应用任务组成，每个应用任务完成特定的工作。
  2. 能否简述嵌入式系统的主要特性？
    • 适应多种处理器、可剪裁、轻量型、实时可靠、可固化
  3. 能否简述嵌入式系统的分类？
    • 按嵌入式处理器的位数来分类：4位、8位、16位、32位、64位
    • 按应用分类：信息家电类、移动终端类、汽车电子类、工业控制类
    • 按速度分类：强实时系统、一般实时系统、弱实时系统
    • 按确定性分类：硬实时、软实时
    • 按嵌入式系统软件复杂程度分类：循环轮询系统、有限状态机系统、前后台系统、单处理器多任务系统、多处理器单任务系统
  4. 能否给出典型的嵌入式系统及其特性？
    • Linux系统：广泛的硬件支持；内核高效稳定；开放源码软件丰富；优秀的开发工具；完善的网络同学和文件管理机制
    • VxWorks系统：高可靠性、高实时性、可裁剪性好，但非常昂贵
    • μC/OS-Ⅱ：开源、结构小、可剥夺、实时的
    • Windows CE：内核较小并具有一定的实时性；实时性不高效率低
    • Symbian：实时性、多任务的纯32位操作系统，功耗低、内存占用少
    • IOS：软硬件兼容性好，性能较强，较流畅
    • QNX：高可靠性、高安全性、分布式、强实时操作系统，高度的伸缩性，可灵活地剪裁

ARM处理器与指令集-内部结构1

  1. 能否分析ARM处理器的主要特点及优势？
    • 高性能、低成本、低功耗
    • 大量的寄存器，都可用于多种用途
    • Load/Store体系结构，非常强大的多寄存器Load和Store指令
    • 3地址指令（两个源操作数寄存器和结果寄存器独立设定）
    • 每条指令都条件执行，包含能在单时钟周期执行的单挑指令内完成一项普通的移位操作和一项普通的ALU操作
    • 能通过协处理器指令集来扩展ARM指令集，包括在编程模式下增加了新的寄存器和数据类型
    • 在Thumb体系结构下以高密度16位压缩形式表示指令集
  2. 能否阐述ARM各个体系结构版本的主要区别及应用场景？
    • v1版架构：
      • 基本的数据处理指令（无乘法）
      • 字节、半字、字的LOAD/STORE指令
      • 转移指令，包括子程序调用及链接指令
      • 软件中断指令
      • 寻址空间：64M字节（26位）
      • 只在原型机ARM1出现过
    • v2版架构：
      • 增加了乘法和乘加指令
      • 支持协处理器操作指令
      • 快速中断模式
      • SWP/SWPB最基本的存储器与寄存器交换指令
      • 寻址空间：64M字节（26位）
      • ARM2与ARM3（V2a版）架构
    • v3版架构
      • 把寻址空间增加至32位，（4G字节）
      • 增加了当前程序状态寄存器CPSR和程序状态保存寄存器SPSR
      • 增加了中止（Abort）和未定义2种处理器模式
      • 指令集增加了MRS、MSR指令，以访问CPSR和SPSR寄存器
      • 增加了从异常处理返回的指令功能
      • ARM6采用了该版架构
    • v4版架构
      • 引进了16位的Thumb指令集
      • 指令集增加了以下功能
        • 符号化和非符号化半字及符号化字节的存取指令
        • 增加了16位的Thumb指令集
        • 完善了软件中断SWI指令的功能
        • 处理器系统模式引进特权方式时使用用户寄存器操作
        • 把一些未使用的指令空间捕获未定义行为
      • 是曾经应用最广的ARM体系结构，对v3版结构进行了进一步扩充
      • ARM7、ARM8、ARM9和Strong ARM都采用该版架构
    • v5版架构：
      • 增加了指令有
        • 带有链接和交换的转移BLX指令
        • 计数前导零CLZ指令，提高了整数除法和重点优先级排队操作的效率
        • BRKT中断指令（软中断）
        • 增加了信号处理指令（V5TE版）
        • 为协处理器增加更多可选择的指令
      • ARM10和XScale都采用该版架构
    • v6版架构
      • 增加多媒体扩展（SIMD变种）功能
      • 在语音、视频、图像上的处理能力提高4倍
      • 使用先进工艺使得芯片的功耗进一步降低
      • ARM11
    • v7版架构
      • 采用了Thumb-2技术，兼容性好
      • 定义了A、R、M三大系列
    • v8版架构
      • 支持64位指令集，包含AArch64和AArch32。AArch64执行状态针对64位处理技术，引入了一个全新的指令集A6
      • 苹果的A7处理器
  3. 能否举例分析ARM微处理器系列芯片的特点？
    • Cortex-M3：基于v7-M架构，采用了纯Thumb2指令的执行方式，具有极高的运算能力和中断相应能力
    • Cortex-R4：基于v7架构，主要目标位产量巨大的高级嵌入式应用系统
    • Cortex-R4F：加入了代码错误矫正技术、浮点运算单元FPU以及DMA综合配置的能力
    • Cortex-A8：运用了可增加代码密度和加强性能的技术、可支持多媒体以及信号处理能力的NEONTM技术、以及能够支持Java和其他文字代码语音的提前和即时编译的Jazelle@RTC技术
    • ARM7：提供Thumb16位压缩指令集和EmbededICE软件调试方式，适用于更大规模的SoC设计中
    • ARM9：兼容ARM7系列且更加灵活
    • ARM9E：允许在硬件中执行Java字节码
    • ARM10E：使用向量浮点VFP单元VFP10提供高性能的浮点解决方案，提高处理器的整型和浮点运算性能
    • Xscale：提供全性能、高性价比、低功耗的解决方案，支持16位Thumb指令并集成数字信号处理DSP指令。

ARM处理器和指令集-内部结构2

  1. 能否分析RISC和CISC的主要特点及区别？

    • 指令集：
      • RISC：一个周期执行一条指令，通过简单操作指令的组合实现复杂操作；指令长度固定。
      • CISC：指令长度不固定，执行需要多个周期
    • 流水线：
      • RISC：流水线每周期前进一步
      • CISC：指令的执行需要调用伪代码的一个微程序
    • 寄存器：
      • RISC：更多通用寄存器
      • CISC：用于特定目的的专用寄存器
    • Load/Store结构：
      • RISC：独立的Load和Store指令完成数据在寄存器和外部存储器之间的传输
      • CISC：处理器能够直接处理存储器中的数据

  2. ARM指令和Thumb指令之间如何切换？

    • 进入Thumb状态：
      • 当操作数寄存器的状态位（位0）为1时，可以采用执行BX指令的方法，使微处理器从ARM状态切换到Thumb状态
      • 此外，当处理器处于Thumb状态时发生异常（如IRQ、FIQ、Undef、Abort、SWI等），则异常处理返回时，自动切换到Thumb状态
    • 进入ARM状态：
      • 当操作数寄存器的状态位为0时，执行BX指令时可以时微处理器从Thumb状态切换到ARM状态。
      • 此外，在处理器进行异常处理时，把PC指针放入异常模式链接寄存器中，并从异常向量地址开始执行程序，也可也使处理器切换到ARM状态。

  3. ARM的七种工作模式的主要用途，及其各种模式下，寄存器的组织情况？

    • 七种工作模式：

      • 用户模式：运行用户应用程序
      • 系统模式：运行操作系统
      • 快中断模式：高速I/O数据传输处理
      • 中断模式：常规I/O数据传输处理
      • 管理模式：开机/复位程序及软中断调用
      • 访问中止：用于支持虚拟内存和/或存储器保护
      • 未定义：自定义指令，协处理器软件仿真

  4. 能否分析状态寄存器的组织结构？

    • CPSR：
      • 4个条件代码标志
      • 2个中断控制位
      • 5个当前处理器模式进行编码的位
      • 1个指示当前指令的工作状态位
      • 保留位
    • SPSR：备份程序状态字，保存异常事件发生之前的CPSR

  5. 能否说明异常模式下，各个部件是如何协调工作的？

    • 对异常的响应
      • 将下一条指令的地址存入相应连接寄存器LR，以便程序在处理异常返回时能从正确的位置重新开始执行。
      • 将CPSR复制到SPSR
      • 根据异常类型，强制设置CPSR的运行模式位
      • 强制PC从相关的异常向量地址取下一条指令执行，从而跳转到相应的异常处理程序处，同时设置中断禁止位，以禁止中断发生。
    • 异常返回
      • 将LR的值减去相应偏移量后送回PC
      • 将SPSR复制回CPSR中
      • 若在进入异常处理时设置了中断禁止位，要在此清除
